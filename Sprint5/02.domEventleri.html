DOM Eventleri - Kapsamlı Örnekler

2.1. DOM Eventleri Nedir?

DOM Eventleri (Olaylar), kullanıcının web sayfası ile etkileşim kurduğunda (tıklama, klavye tuşlarına basma, fare hareketi vb.) tarayıcının oluşturduğu sinyallerdir. JavaScript ile bu olayları dinleyip, kullanıcı etkileşimlerine anında yanıt verebiliriz.

Event'lerin Önemi

Modern web uygulamaları, kullanıcı etkileşimlerine anında yanıt vermelidir. Event'ler sayesinde:

  • Kullanıcı tıklamalarını yakalayabiliriz
  • Form gönderimlerini kontrol edebiliriz
  • Klavye girdilerini işleyebiliriz
  • Fare hareketlerini takip edebiliriz
  • Sayfa yükleme durumlarını yönetebiliriz
  • Dinamik içerik oluşturabiliriz

Event Türleri Kategorileri

1. Mouse Event'leri (Fare Olayları):

  • click - Elemente tıklama (en yaygın kullanılan)
  • dblclick - Çift tıklama
  • mousedown - Fare tuşuna basma
  • mouseup - Fare tuşunu bırakma
  • mouseenter - Fare elementin üzerine gelme
  • mouseleave - Fare elementten ayrılma
  • mouseover - Fare elementin üzerinde (bubbling yapar)
  • mouseout - Fare elementten çıkma (bubbling yapar)
  • mousemove - Fare hareket ettirme

2. Keyboard Event'leri (Klavye Olayları):

  • keydown - Tuşa basma (en yaygın)
  • keyup - Tuşu bırakma
  • keypress - Tuşa basma (deprecated, keydown kullanın)

3. Form Event'leri:

  • input - Input değeri değiştiğinde (anlık)
  • change - Değer değişip focus kaybedildiğinde
  • submit - Form gönderildiğinde
  • focus - Elemente odaklanıldığında
  • blur - Elementten odak ayrıldığında
  • select - Input içeriği seçildiğinde

4. Window/Document Event'leri:

  • load - Sayfa/element yüklendiğinde
  • DOMContentLoaded - DOM yüklendiğinde (daha hızlı)
  • resize - Pencere boyutu değiştiğinde
  • scroll - Sayfa kaydırıldığında
  • beforeunload - Sayfa kapatılmadan önce

5. Touch Event'leri (Mobil):

  • touchstart - Dokunma başladığında
  • touchend - Dokunma bittiğinde
  • touchmove - Dokunma hareket ettiğinde

2.2. addEventListener - Event Dinleme

addEventListener() metodu, bir elemente event dinleyici eklemek için kullanılır. Bu metod, modern JavaScript'te event eklemenin önerilen yöntemidir.

Syntax (Sözdizimi):

element.addEventListener("event", function, useCapture);

Parametreler:

  • event (string): Dinlenecek event adı (örn: "click", "mouseenter")
  • function (function): Event gerçekleştiğinde çalışacak fonksiyon
  • useCapture (boolean, opsiyonel): Event capturing kullanılsın mı? (varsayılan: false)

addEventListener'ın Avantajları:

  • ✅ Aynı event'e birden fazla listener eklenebilir
  • ✅ removeEventListener ile kolayca kaldırılabilir
  • ✅ useCapture parametresi ile event propagation kontrolü
  • ✅ HTML'den ayrı tutulur (separation of concerns)
  • ✅ Daha okunabilir ve bakımı kolay kod

Eski Yöntem vs Yeni Yöntem:

❌ Eski Yöntem (Önerilmez):

<button onclick="myFunction()">Tıkla</button>
// veya
button.onclick = function() { ... };

✅ Yeni Yöntem (Önerilen):

button.addEventListener("click", function() { ... });

Pratik Örnek:

Butona tıklandı! ✅ Event listener başarıyla çalıştı.

Kod Örneği:

const button = document.getElementById("clickBtn");
button.addEventListener("click", function() {
    console.log("Butona tıklandı!");
    // İşlemler burada yapılır
});

2.3. Click Event Örnekleri

click event'i, web geliştirmede en sık kullanılan event türüdür. Kullanıcı bir elemente tıkladığında tetiklenir.

Kullanım Senaryoları:

  • Buton tıklamaları
  • Menü açma/kapama
  • Modal pencereler
  • Form gönderimi
  • Toggle işlemleri (aç/kapa)
  • Sayfa navigasyonu

Örnek 1: Sayaç (Counter)

Her tıklamada sayacı artıran basit bir örnek. Bu pattern, oyun skorları, beğeni sayıları gibi durumlarda kullanılır. 

let count = 0;
button.addEventListener("click", function() {
    count++;
    counter.textContent = count;
});

Örnek 2: Renk Değiştirme

Her tıklamada farklı bir renge geçiş yapan dinamik örnek. Tema değiştirme, renk seçici gibi uygulamalarda kullanılır.

Renkli Kutu
const colors = ["#6366f1", "#8b5cf6", "#a855f7", ...];
let index = 0;
button.addEventListener("click", function() {
    index = (index + 1) % colors.length;
    box.style.background = colors[index];
});

Örnek 3: Gizle/Göster (Toggle)

Bir elementi görünür/görünmez yapma işlemi. Dropdown menüler, açıklama panelleri gibi durumlarda yaygın kullanılır.

Bu kutu gizlenebilir 
button.addEventListener("click", function() {
    if (box.style.display === "none") {
        box.style.display = "flex";
    } else {
        box.style.display = "none";
    }
});

Best Practices (En İyi Uygulamalar):

  • ✅ Event listener'ları sayfa yüklendikten sonra ekleyin
  • ✅ Gereksiz event listener'ları kaldırın (memory leak önleme)
  • ✅ Event delegation kullanarak performansı artırın
  • ✅ preventDefault() ile varsayılan davranışları kontrol edin

2.4. Mouse Event Örnekleri

Mouse event'leri, kullanıcının fare ile yaptığı tüm etkileşimleri yakalar. Modern web uygulamalarında hover efektleri, drag & drop, context menu gibi özellikler için kritik öneme sahiptir.

Mouse Event Türleri:

  • click - Tıklama (mousedown + mouseup kombinasyonu)
  • mousedown - Fare tuşuna basma
  • mouseup - Fare tuşunu bırakma
  • mouseenter - Fare elementin içine girer (bubbling yapmaz)
  • mouseleave - Fare elementten çıkar (bubbling yapmaz)
  • mouseover - Fare elementin üzerinde (bubbling yapar)
  • mouseout - Fare elementten çıkar (bubbling yapar)
  • mousemove - Fare hareket eder
  • contextmenu - Sağ tıklama menüsü

mouseenter vs mouseover Farkı:

mouseenter sadece hedef elementte tetiklenir (bubbling yok), mouseover ise child elementlerde de tetiklenir (bubbling var). Genellikle mouseenter/mouseleave tercih edilir.

Örnek 1: mouseenter ve mouseleave

Hover efektleri, tooltip gösterme, görsel büyütme gibi işlemlerde kullanılır.

Fareyi üzerime getir 
box.addEventListener("mouseenter", function() {
    // Fare kutu üzerine geldiğinde
    box.style.transform = "scale(1.15)";
});

box.addEventListener("mouseleave", function() {
    // Fare kutudan ayrıldığında
    box.style.transform = "scale(1)";
});

Örnek 2: mousedown ve mouseup

Drag & drop, resizing, drawing gibi işlemlerde kullanılır. Kullanıcı fare tuşunu basılı tuttuğu süre boyunca işlem yapılabilir.

Basılı tut 
box.addEventListener("mousedown", function() {
    // Tuşa basıldığında
    box.style.transform = "scale(0.9)";
});

box.addEventListener("mouseup", function() {
    // Tuş bırakıldığında
    box.style.transform = "scale(1)";
});

Örnek 3: mousemove ile Pozisyon Takibi

Fare pozisyonunu takip ederek interaktif özellikler oluşturulabilir.

Fareyi hareket ettir

Kullanım Senaryoları:

  • Hover efektleri ve animasyonlar
  • Tooltip ve popover gösterme
  • Drag & drop işlemleri
  • Resim galerilerinde zoom
  • Oyun kontrolleri
  • Çizim uygulamaları

2.5. Keyboard Event Örnekleri

Klavye event'leri, kullanıcının klavye ile yaptığı girdileri yakalar. Form validasyonu, kısayol tuşları, arama özellikleri gibi durumlarda kritik öneme sahiptir.

Keyboard Event Türleri:

  • keydown - Tuşa basıldığında (en yaygın, önerilen)
  • keyup - Tuş bırakıldığında
  • keypress - Tuşa basıldığında (deprecated, kullanmayın)

Event Object Özellikleri:

  • e.key - Basılan tuşun değeri (örn: "a", "Enter", "ArrowUp")
  • e.keyCode - Tuşun kod numarası (deprecated, key kullanın)
  • e.code - Fiziksel tuş kodu (örn: "KeyA", "Enter")
  • e.ctrlKey - Ctrl tuşu basılı mı? (boolean)
  • e.shiftKey - Shift tuşu basılı mı? (boolean)
  • e.altKey - Alt tuşu basılı mı? (boolean)
  • e.metaKey - Meta tuşu basılı mı? (Mac'te Cmd) (boolean)

Örnek 1: keydown ile Tuş Bilgileri

Her tuş basımında tuş bilgilerini gösterir. Kısayol tuşları, oyun kontrolleri gibi durumlarda kullanılır. 

input.addEventListener("keydown", function(e) {
    console.log("Tuş:", e.key);
    console.log("Kod:", e.code);
    console.log("Ctrl:", e.ctrlKey);
    console.log("Shift:", e.shiftKey);
});

Örnek 2: Enter Tuşu ile Gönder

Form gönderimi, arama, mesaj gönderme gibi işlemlerde Enter tuşu ile hızlı işlem yapılabilir. 

input.addEventListener("keyup", function(e) {
    if (e.key === "Enter") {
        // Enter tuşuna basıldı
        console.log("Gönder:", e.target.value);
    }
});

Örnek 3: Kısayol Tuşları (Keyboard Shortcuts)

Ctrl+S (kaydet), Ctrl+C (kopyala) gibi kısayol tuşları oluşturulabilir. 

document.addEventListener("keydown", function(e) {
    // Ctrl + S (Kaydet)
    if (e.ctrlKey && e.key === "s") {
        e.preventDefault(); // Tarayıcının kaydetme menüsünü engelle
        saveFunction();
    }
    
    // Escape tuşu
    if (e.key === "Escape") {
        closeModal();
    }
});

Özel Tuşlar:

  • Enter - Form gönderimi
  • Escape - Modal kapatma, iptal
  • ArrowUp/Down/Left/Right - Navigasyon
  • Tab - Form alanları arası geçiş
  • Backspace/Delete - Silme işlemleri

Best Practices:

  • ✅ keydown kullanın, keypress kullanmayın (deprecated)
  • ✅ e.key kullanın, e.keyCode kullanmayın (deprecated)
  • ✅ preventDefault() ile varsayılan davranışları kontrol edin
  • ✅ Kısayol tuşlarını dokümente edin

2.6. Form Event Örnekleri

Form event'leri, kullanıcı form alanları ile etkileşim kurduğunda tetiklenir. Form validasyonu, anlık geri bildirim, otomatik kaydetme gibi özellikler için kritiktir.

Form Event Türleri:

  • input - Input değeri her değiştiğinde (anlık, her karakterde)
  • change - Değer değişip focus kaybedildiğinde
  • submit - Form gönderildiğinde
  • focus - Elemente odaklanıldığında
  • blur - Elementten odak ayrıldığında
  • select - Input içeriği seçildiğinde
  • reset - Form sıfırlandığında

input vs change Farkı:

input event'i her karakter değişikliğinde tetiklenir (anlık). change event'i ise değer değişip focus kaybedildiğinde tetiklenir.

Örnek 1: input Event - Anlık Değişiklik Takibi

Karakter sayısı, şifre gücü, arama önerileri gibi anlık geri bildirim gereken durumlarda kullanılır.

Karakter sayısı: 0
input.addEventListener("input", function(e) {
    const length = e.target.value.length;
    result.textContent = `Karakter sayısı: ${length}`;
    // Her karakter değişikliğinde çalışır
});

Örnek 2: change Event - Değer Değiştiğinde

Select, checkbox, radio button gibi elementlerde değer değişikliğini yakalamak için kullanılır. 

select.addEventListener("change", function(e) {
    console.log("Seçilen değer:", e.target.value);
    // Focus kaybedildiğinde çalışır
});

Örnek 3: submit Event - Form Gönderimi

Form gönderilmeden önce validasyon yapma, veri işleme, AJAX ile gönderme gibi işlemler için kullanılır. 

form.addEventListener("submit", function(e) {
    e.preventDefault(); // Sayfa yenilenmesini engelle
    
    // Validasyon
    const name = formName.value;
    const email = formEmail.value;
    
    if (name && email) {
        // Form gönder
        console.log("Gönderiliyor:", { name, email });
    }
});

Örnek 4: focus ve blur Event'leri

Input alanına odaklanıldığında veya odaktan çıkıldığında işlem yapılabilir. 

input.addEventListener("focus", function() {
    // Odaklandığında
    this.style.borderColor = "#8b5cf6";
    this.style.boxShadow = "0 0 0 4px rgba(139, 92, 246, 0.1)";
});

input.addEventListener("blur", function() {
    // Odaktan çıkıldığında
    this.style.borderColor = "#e0e0e0";
    // Validasyon yapılabilir
    validateInput(this);
});

Form Validasyon Örneği:

function validateEmail(email) {
    const regex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    return regex.test(email);
}

emailInput.addEventListener("blur", function() {
    if (!validateEmail(this.value)) {
        this.style.borderColor = "#f43f5e";
        showError("Geçerli bir e-posta adresi girin");
    } else {
        this.style.borderColor = "#8b5cf6";
    }
});

Best Practices:

  • ✅ submit event'inde preventDefault() kullanın (AJAX için)
  • ✅ input event'i için debounce kullanın (performans)
  • ✅ blur event'inde validasyon yapın
  • ✅ Kullanıcıya anlık geri bildirim verin
  • ✅ Form verilerini güvenli şekilde işleyin

2.7. Event Object ve Özellikleri

Event listener fonksiyonlarına otomatik olarak bir Event Object parametresi geçirilir. Bu obje, event hakkında tüm önemli bilgileri içerir ve event'i kontrol etmemize olanak sağlar.

Event Object Nedir?

Her event listener fonksiyonu, ilk parametre olarak bir Event objesi alır. Bu obje, event hakkında detaylı bilgiler içerir ve event'in davranışını kontrol etmemize izin verir.

Temel Event Object Özellikleri:

  • e.type - Event türü (örn: "click", "keydown")
  • e.target - Event'in tetiklendiği element
  • e.currentTarget - Event listener'ın eklendiği element
  • e.timeStamp - Event'in oluştuğu zaman (milisaniye)
  • e.bubbles - Event bubbling yapıyor mu? (boolean)
  • e.cancelable - Event iptal edilebilir mi? (boolean)
  • e.defaultPrevented - preventDefault() çağrıldı mı? (boolean)

target vs currentTarget Farkı:

target: Event'in gerçekten tetiklendiği element (child element olabilir)
currentTarget: Event listener'ın eklendiği element (her zaman aynı) 

div.addEventListener("click", function(e) {
    console.log(e.target);        // Tıklanan element (button olabilir)
    console.log(e.currentTarget); // Her zaman div
});

Mouse Event Object Özellikleri:

  • e.clientX / e.clientY - Mouse pozisyonu (viewport'a göre)
  • e.pageX / e.pageY - Mouse pozisyonu (sayfaya göre)
  • e.screenX / e.screenY - Mouse pozisyonu (ekrana göre)
  • e.button - Hangi fare tuşu basıldı? (0: sol, 1: orta, 2: sağ)
  • e.buttons - Basılı tuşların kombinasyonu

Mouse Pozisyonu Örneği:

Fareyi hareket ettir 
box.addEventListener("mousemove", function(e) {
    const rect = box.getBoundingClientRect();
    const x = e.clientX - rect.left;  // Kutu içindeki X pozisyonu
    const y = e.clientY - rect.top;   // Kutu içindeki Y pozisyonu
    console.log(`X: ${x}, Y: ${y}`);
});

Keyboard Event Object Özellikleri:

  • e.key - Basılan tuşun değeri ("a", "Enter", "ArrowUp")
  • e.code - Fiziksel tuş kodu ("KeyA", "Enter")
  • e.keyCode - Tuş kodu (deprecated)
  • e.ctrlKey - Ctrl tuşu basılı mı?
  • e.shiftKey - Shift tuşu basılı mı?
  • e.altKey - Alt tuşu basılı mı?
  • e.metaKey - Meta tuşu basılı mı? (Mac: Cmd)
  • e.repeat - Tuş basılı tutuldu mu? (boolean)

Event Object Metodları:

  • e.preventDefault() - Varsayılan davranışı engelle
  • e.stopPropagation() - Event'in yayılmasını durdur
  • e.stopImmediatePropagation() - Aynı elementteki diğer listener'ları da durdur

Kullanım Senaryoları:

  • Mouse pozisyonunu takip etme (drag & drop, drawing)
  • Hangi tuşa basıldığını öğrenme (kısayol tuşları)
  • Event'in nereden geldiğini bulma (event delegation)
  • Varsayılan davranışları kontrol etme (form gönderimi)

2.8. Event Propagation (Bubbling ve Capturing)

Event Propagation, bir event'in DOM ağacında nasıl yayıldığını açıklar. Bu kavramı anlamak, karmaşık event yönetimi için kritik öneme sahiptir.

Event Propagation Fazları:

Bir event, DOM ağacında 3 fazda işlenir:

  1. Capturing Phase (Yakalama Fazı) - Event, root'tan hedefe doğru iner
  2. Target Phase (Hedef Fazı) - Event, hedef elementte işlenir
  3. Bubbling Phase (Kabarcık Fazı) - Event, hedeften root'a doğru çıkar

Bubbling (Kabarcık) Nedir?

Bubbling, event'in hedef elementten başlayarak parent elementlere doğru yayılmasıdır. Varsayılan olarak çoğu event bubbling yapar. 

<div id="outer">
    <div id="inner">
        <button>Tıkla</button>
    </div>
</div>

// Button'a tıklandığında:
// 1. button (target)
// 2. inner (bubbling)
// 3. outer (bubbling)
// 4. body (bubbling)
// 5. html (bubbling)
// 6. document (bubbling)

Capturing (Yakalama) Nedir?

Capturing, event'in root'tan başlayarak hedefe doğru inmesidir. useCapture parametresini true yaparak capturing kullanılabilir. 

element.addEventListener("click", function() {
    // Bubbling fazında çalışır (varsayılan)
}, false);

element.addEventListener("click", function() {
    // Capturing fazında çalışır
}, true);

Pratik Örnek:

Dış Kutu (Tıklayın)
İç Kutu (Tıklayın)

stopPropagation() Kullanımı:

e.stopPropagation() ile event'in yayılmasını durdurabiliriz. Bu, sadece mevcut fazdaki yayılmayı durdurur. 

button.addEventListener("click", function(e) {
    e.stopPropagation(); // Event'in yukarı yayılmasını durdur
    console.log("Sadece bu çalışır");
});

stopImmediatePropagation() Kullanımı:

e.stopImmediatePropagation() ile aynı elementteki diğer listener'ların da çalışmasını engelleyebiliriz. 

button.addEventListener("click", function(e) {
    e.stopImmediatePropagation(); // Diğer listener'ları da durdur
});

Kullanım Senaryoları:

  • Modal pencereler (dışarı tıklamada kapatma)
  • Dropdown menüler (dışarı tıklamada kapatma)
  • Event delegation (parent'ta dinleme)
  • Özel kontroller (bazı event'leri durdurma)

Best Practices:

  • ✅ Çoğu durumda bubbling yeterlidir
  • ✅ stopPropagation()'ı dikkatli kullanın (diğer listener'ları etkileyebilir)
  • ✅ Event delegation için bubbling'i kullanın
  • ✅ Modal/dropdown için dışarı tıklama kontrolü yapın

2.9. removeEventListener - Event Dinleyiciyi Kaldırma

Bir event listener'ı kaldırmak için removeEventListener() kullanılır. Bu, memory leak'leri önlemek ve gereksiz event listener'ları temizlemek için önemlidir.

Syntax:

element.removeEventListener("event", function, useCapture);

Önemli Kural:

Kaldırılacak fonksiyon, eklenen fonksiyonla aynı referansa sahip olmalıdır. Bu yüzden fonksiyonu bir değişkende saklamak gerekir.

❌ Yanlış Kullanım:

button.addEventListener("click", function() {
    console.log("Tıklandı");
});

// Bu çalışmaz! Fonksiyon referansı farklı
button.removeEventListener("click", function() {
    console.log("Tıklandı");
});

✅ Doğru Kullanım:

// Fonksiyonu değişkende sakla
function handleClick() {
    console.log("Tıklandı");
}

button.addEventListener("click", handleClick);
button.removeEventListener("click", handleClick); // ✅ Çalışır

Pratik Örnek:

Kod Örneği:

// Fonksiyonu dışarıda tanımla
function testButtonHandler() {
    console.log("Test butonu tıklandı!");
}

// Ekle
addListenerBtn.addEventListener("click", function() {
    testBtn.addEventListener("click", testButtonHandler);
});

// Kaldır
removeListenerBtn.addEventListener("click", function() {
    testBtn.removeEventListener("click", testButtonHandler);
});

Ne Zaman Kaldırılmalı?

  • ✅ Element DOM'dan kaldırıldığında
  • ✅ Component unmount olduğunda (React, Vue vb.)
  • ✅ Geçici event listener'lar kullanıldığında
  • ✅ Memory leak önlemek için
  • ✅ Event listener'ı tekrar eklemek istediğinizde

Arrow Function ile Dikkat:

// Arrow function kullanırken de referansı saklamalısınız
const handleClick = () => {
    console.log("Tıklandı");
};

button.addEventListener("click", handleClick);
button.removeEventListener("click", handleClick); // ✅ Çalışır

Best Practices:

  • ✅ Event listener'ları eklerken referansı saklayın
  • ✅ Component cleanup'ında listener'ları kaldırın
  • ✅ Gereksiz listener'ları temizleyin (performans)
  • ✅ addEventListener ile eklenen listener'ları removeEventListener ile kaldırın

2.10. Event Delegation (Event Temsilciliği)

Event Delegation, event listener'ları child elementler yerine parent element'e ekleme tekniğidir. Bu, özellikle dinamik olarak eklenen elementler için performans ve kod organizasyonu açısından çok faydalıdır.

Event Delegation Nedir?

Event delegation, event bubbling özelliğinden yararlanarak, her child element'e ayrı ayrı listener eklemek yerine, parent element'e tek bir listener ekleme tekniğidir.

❌ Normal Yöntem (Her Elemente Listener):

// Her butona ayrı listener ekleme
const buttons = document.querySelectorAll(".deleteBtn");
buttons.forEach(button => {
    button.addEventListener("click", function() {
        this.parentElement.remove();
    });
});

// Problem: Yeni eklenen butonlar için çalışmaz!

✅ Event Delegation Yöntemi:

// Parent'a tek listener ekleme
const list = document.getElementById("itemList");
list.addEventListener("click", function(e) {
    // Tıklanan element deleteBtn mi kontrol et
    if (e.target.classList.contains("deleteBtn")) {
        e.target.parentElement.remove();
    }
});

// Avantaj: Yeni eklenen butonlar da otomatik çalışır!

Event Delegation'un Avantajları:

  • ✅ Performans: Tek bir listener, birden fazla element için çalışır
  • ✅ Dinamik Elementler: Sonradan eklenen elementler için de çalışır
  • ✅ Memory: Daha az memory kullanımı
  • ✅ Kod Organizasyonu: Daha temiz ve bakımı kolay kod
  • ✅ Esneklik: Element yapısı değişse bile çalışır

e.target Kullanımı:

Event delegation'da e.target ile hangi child element'in tıklandığını öğrenebiliriz. 

parent.addEventListener("click", function(e) {
    // e.target: Gerçekten tıklanan element
    // e.currentTarget: Listener'ın eklendiği element (parent)
    
    if (e.target.classList.contains("deleteBtn")) {
        // Sil butonuna tıklandı
    } else if (e.target.tagName === "LI") {
        // Liste öğesine tıklandı
    }
});

Kullanım Senaryoları:

  • Dinamik liste öğeleri (todo list, shopping cart)
  • Tab menüleri
  • Modal pencereler
  • Form alanları (dinamik eklenen)
  • Büyük tablolarda satır tıklamaları

Best Practices:

  • ✅ Parent element'e mümkün olduğunca yakın ekleyin
  • ✅ e.target ile doğru element kontrolü yapın
  • ✅ closest() metodu ile parent element bulun
  • ✅ Event delegation'u gereksiz yere kullanmayın (basit durumlarda normal listener yeterli)

closest() Metodu ile Gelişmiş Kullanım:

list.addEventListener("click", function(e) {
    // En yakın .item class'ına sahip elementi bul
    const item = e.target.closest(".item");
    if (item) {
        // item içindeki herhangi bir yere tıklandı
        item.remove();
    }
});

2.11. preventDefault() ve stopPropagation()

Event Object'in iki önemli metodu, event davranışını kontrol etmemize olanak sağlar.

preventDefault() Nedir?

e.preventDefault(), event'in varsayılan davranışını engeller. Örneğin, bir link'e tıklandığında sayfa yönlendirmesini engelleyebiliriz.

preventDefault() Kullanım Senaryoları:

  • Form gönderimi: Sayfa yenilenmesini engellemek (AJAX için)
  • Link tıklaması: Sayfa yönlendirmesini engellemek
  • Kısayol tuşları: Tarayıcı kısayollarını engellemek (Ctrl+S)
  • Context menu: Sağ tıklama menüsünü engellemek

Örnekler:

// Form gönderimini engelle
form.addEventListener("submit", function(e) {
    e.preventDefault();
    // AJAX ile gönder
});

// Link tıklamasını engelle
link.addEventListener("click", function(e) {
    e.preventDefault();
    // Özel işlem yap
});

// Kısayol tuşunu engelle
document.addEventListener("keydown", function(e) {
    if (e.ctrlKey && e.key === "s") {
        e.preventDefault(); // Tarayıcının kaydetme menüsünü engelle
        customSave();
    }
});

stopPropagation() Nedir?

e.stopPropagation(), event'in DOM ağacında yayılmasını durdurur. Bubbling veya capturing fazında event'in parent elementlere ulaşmasını engeller.

stopPropagation() Kullanım Senaryoları:

  • Modal pencereler (dışarı tıklamada kapatma kontrolü)
  • Dropdown menüler (dışarı tıklamada kapatma)
  • Nested elementler (iç element tıklamasında dış elementin tetiklenmemesi)

Örnek:

button.addEventListener("click", function(e) {
    e.stopPropagation(); // Parent elementlerdeki listener'lar çalışmaz
    console.log("Sadece bu çalışır");
});

// stopImmediatePropagation() - Aynı elementteki diğer listener'ları da durdurur
button.addEventListener("click", function(e) {
    e.stopImmediatePropagation();
    console.log("Sadece bu çalışır, aynı elementteki diğer listener'lar çalışmaz");
});

İkisini Birlikte Kullanma:

button.addEventListener("click", function(e) {
    e.preventDefault();      // Varsayılan davranışı engelle
    e.stopPropagation();    // Event yayılmasını durdur
    // Özel işlem
});

2.12. Özet ve Best Practices

DOM Eventleri Özeti:

  • ✅ addEventListener kullanın (onclick yerine)
  • ✅ Event listener'ları sayfa yüklendikten sonra ekleyin
  • ✅ Gereksiz listener'ları kaldırın (memory leak önleme)
  • ✅ Event delegation kullanın (dinamik elementler için)
  • ✅ preventDefault() ile varsayılan davranışları kontrol edin
  • ✅ stopPropagation()'ı dikkatli kullanın

Event Türleri Özeti:

Event Türü Kullanım
click Buton tıklamaları, menü açma
mouseenter/leave Hover efektleri, tooltip
keydown/keyup Klavye girdileri, kısayol tuşları
input Anlık form değişiklikleri
submit Form gönderimi
focus/blur Input odak yönetimi

Yaygın Hatalar ve Çözümleri:

  • ❌ Hata: HTML'de onclick kullanmak
    ✅ Çözüm: addEventListener kullanın
  • ❌ Hata: removeEventListener'da fonksiyon referansı hatası
    ✅ Çözüm: Fonksiyonu değişkende saklayın
  • ❌ Hata: Dinamik elementlere listener eklemek
    ✅ Çözüm: Event delegation kullanın
  • ❌ Hata: Gereksiz event listener'ları kaldırmamak
    ✅ Çözüm: Component cleanup'ında kaldırın

Performans İpuçları:

  • ✅ Event delegation kullanın (büyük listeler için)
  • ✅ Debounce/throttle kullanın (sık tetiklenen event'ler için)
  • ✅ Gereksiz listener'ları kaldırın
  • ✅ Passive event listener'ları kullanın (scroll için)
